ssh:----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
	ssh putty plink 三个软件都是一个，只是平台不一样：
		ssh:linux系统已经集成了的，用来远程登录		sshd
		putty:windows平台下ssh远程登录工具
		plink：Windows平台下的putty命令行版本
	
	windows下的服务端是openssh，Linux下的是sshd,
	客户端配置文件：/etc/ssh/ssh_config
	服务端配置文件：/etc/ssh/sshd_config

	正向代理和反向代理的区别：正向代理的对象是客户端（正向代理client对server是透明的），反向代理的对象是内网服务器（server对client是透明的，比如nginx）

	SSH和plink做代理的时候，它们是有区别的：
		ssh没有-pw的选项指定密码，所以需要交互，plink可以使用-pw避免交互输入密码
		ssh和plink默认都会提示验证远程主机的指纹，提示用户是否要连接，要避免这个交互，plink可以使用echo这个系统命令，echo 一个 'yes'，通过管道符号交给plink，而ssh添加一个选项就可以了：-o stricthostkeychecking=no 意思是不检查主机的指纹

	-L Local的缩写，意思是把本地的端口转发到远程服务器上（也叫正向代理代理）
		ssh -L 本地IP:本地端口:远程IP:远程端口 远程的系统账号@远程IP	如果本地IP不填写，默认是127.0.0.1
		例：ssh -L 192.168.1.1:1111:192.168.1.2:2222 test@192.168.1.2
			意思是：把本地192.168.1.1的1111端口映射到192.168.1.2:的2222端口，访问192.168.1.1的1111端口，就是访问192.168.1.2的2222端口
	-R Remote的缩写，意思是把远程的端口转发到本地（也叫反向代理代理）一般的应用场景是把服务端不能被外界访问的端口代理到本地
		ssh -R [远程IP]:远程IP的端口:本地IP:本地IP的端口 远程系统账号@远程IP	[远程IP]不管填写不填写，最后都是监听127.0.0.1，那就需要修改sshd_config配置文件，把GatewayPorts修改成yes
		例：ssh -R 192.168.1.2:2222:192.168.1.1:1111 account@192.168.1.2
			意思是：把本地192.168.1.1的端口1111，映射到远程IP192.168.1.2的2222端口上，远程192.168.1.2访问自己的2222就是访问192.168.1.1的1111端口
	socket5:
		1:只需要客户端执行就好：ssh -N -D localport remove-user@remote-ip -p remote-ssh-port
		2:设置浏览器的代理为127.0.0.1 localport

	plink代理：
		正向代理：把本地的端口转发到远程，连接本地的端口，就是连接远程的端口
			echo 'yes'|plink.exe -C -N -L 127.0.0.1:3389:192.168.0.66:3389 -pw 123456 test@192.168.0.66	含义：本机使用test帐号，密码123456，以不执行命令的方式(-N)连接到远程服务器192.168.0.66的ip，把连接本机127.0.0.1的3389端口的数据，转发到远程服务器192.168.0.66的3389端口上
		反向代理：把远程的端口转发到本地，连接远程的端口，就是连接本地的端口
			echo 'yes'|plink.exe -C -N -R 192.168.0.66:3389:127.0.0.1:3389 -pw 123456 test@192.168.0.66 -P 27786	含义：本机使用test帐号，密码123456，以不执行命令的方式(-N)连接到远程服务器192.168.0.66的ip，把连接远程服务器192.168.0.66的3389端口的数据，转发到本机127.0.0.1的3389端口上
			echo 'yes'|plink-x64.exe -C -N -R 127.0.0.1:3389:127.0.0.1:3389 -pw qweasd123 test@外网ip -P 22

		技巧：从哪里转发到哪里，会比较迷糊，简单点，不管正向连接，反向连接，都可以这个样子来理解
			比如：127.0.0.1:3389:192.168.0.66:3389
				分为前后两节：127.0.0.1:3389	192.168.0.66:3389	把前面的，转发到后面
